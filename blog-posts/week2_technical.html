Title: What are the differences between relative, absolute, and fixed positioning?

Date:9/6/14

HTML elements may be positioned in several specific ways: absolute, fixed, relative, and the default, static. Let's go over each so we may see how they differ.

Static

Starting with the default, static, is the normal default position. This usually does not have to be set. (If it needed to be set, the code example is below.)

Example: position:static;

Relative

Example: position: relative: top: 10px.

Relative positioning allows a developer to shift an element from its normal, static position. When using relative positioning, you need to specify where to shift and how much. (Note that, by setting relative position, the element will on top of any static elements it overlaps.)

Absolute

Example: {position:absolute;top:0;left:0;}

Absolute positioning allows a developer to place an element irrespective of random elements on the page. Its positioning on the page is only constrained relative to the next parents element with relative or absolute positioning, or html element if none other such elements exist.

Fixed

[Hard to show in a picture. Position is fixed and remains in place even when scrolling the window.]

Example: {position:fixed;top:0;left:0}

Fixed positioning positions element relative to the actual page, staying fixed even when the window is scrolled.
