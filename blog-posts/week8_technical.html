Title: Big O Complexity and Notation

Date: 11/19/14

Big O notation is used to measure the performance/complexity of an algorithm by how the algorithm responds to different input sizes. Big O notation often signifies the algorithm's execution time or the memory space used. An algorithm is measured with respect to the worst-possible response for a particular input size.

The following are examples of Big O notation describing algorithms:

  • O(1) describes an algorithm that executes any input size in the same time or same amount of space.
  • O(N) describes an algorithm that adjusts linearly in proportion to the size of the input.
  • O(N^2) represents an algorithm that responds parabolically to input size.
  • O(2^N) describes an algorithm that takes double time or space per increase in input size. Increases in input size will quickly increase the time and space required.
  • Other relationships, like O(log N), may also be cited to represent other algorithm relationships to input size.
